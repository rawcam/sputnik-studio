Общее описание

Sputnik Studio — профессиональное веб-приложение для инженерного проектирования аудиовизуальных систем, систем видеоконференцсвязи, LED-дисплеев, сетевой инфраструктуры и эргономики рабочих мест. Инструмент позволяет выполнять комплексные расчёты, сохранять проекты в браузере, экспортировать их в JSON и формировать печатные отчёты.

Приложение построено на модульной архитектуре с единым хранилищем состояния. Каждый раздел (видео, сеть, LED, звук, ВКС, тракты, эргономика, энергия) содержит независимые калькуляторы, которые обмениваются данными через общее хранилище. Это позволяет создавать сквозные проекты, где параметры одного раздела автоматически влияют на смежные расчёты.

Интерфейс адаптирован для работы на настольных компьютерах, планшетах и смартфонах. Доступна тёмная тема, сворачиваемая боковая панель, плавная навигация по разделам. Все вычисления производятся на стороне клиента, данные не покидают устройство пользователя.

Основные возможности:

  • Расчёт пропускной способности видеопотоков с учётом кодека, разрешения, частоты кадров, цветовой субдискретизации и битности.
  • Моделирование сетевой нагрузки с поддержкой QoS, Multicast, PoE и расчётом суммарного битрейта.
  • Проектирование LED-экранов в двух режимах: по кабинетам (с округлением пикселей вниз) и по разрешению. Автоматический расчёт физических размеров, разрешения, площади, мощности и минимального расстояния просмотра.
  • Акустические расчёты: уровень звукового давления (SPL), время реверберации (RT60) по формуле Сэбина, подбор потолочных громкоговорителей.
  • Оценка требований к видеоконференцсвязи (битрейт для P2P и MCU-конференций) с учётом популярных платформ и кодеков.
  • Построение сквозных трактов сигнала с учётом задержек, PoE-бюджета и энергопотребления устройств.
  • Эргономический анализ зон видимости экранов коллективного пользования по методике угловых размеров (α = 0.25°, 0.38°, 0.55°).
  • Новое в версии 6.1.0: модуль «Энергия» для подбора ИБП (с учётом типа ИБП, напряжения и типа батарей, пусковых токов) и кондиционера (с учётом площади и типа помещения).
  • Новое в версии 6.1.0: встроенный логгер с парольной защитой, собирающий информацию о действиях пользователя, состоянии сети, ошибках; позволяет экспортировать логи и анализировать работу системы.
  • Управление проектом: автосохранение в localStorage, экспорт/импорт JSON, печать отчёта, сброс данных.

Необходимая минимальная квалификация инженера

Для эффективного использования Sputnik Studio и корректной интерпретации результатов расчётов специалист должен обладать следующими компетенциями:

  • Видеотехника: глубокое понимание цифровых видеопотоков (разрешения, кадровая частота, цветовые пространства RGB/YCbCr, субдискретизация 4:4:4/4:2:2/4:2:0, битность 8/10/12). Знание кодеков H.264, H.265, MJPEG, SDVoE, NDI, их областей применения и влияния на задержку и битрейт. Умение рассчитывать несжатый битрейт по формуле B = W×H×FPS×bitDepth×channels.
  • Сетевые технологии: знание стандартов Ethernet (10/100/1000/10G), принципов коммутации, VLAN (802.1Q), IGMP Snooping для оптимизации Multicast, QoS (DiffServ, 802.1p), PoE (802.3af/at/bt) с расчётом бюджетов, протоколов синхронизации PTP (IEEE 1588) и NTP. Опыт настройки управляемых коммутаторов для AV-over-IP.
  • LED-дисплеи: понимание шага пикселя (pitch), типов кабинетов, расчёта физических размеров и разрешения, требований к минимальному расстоянию просмотра (L_min = 1000 × pitch (мм)), оценки мощности и тепловыделения, калибровки яркости и цвета.
  • Акустика и звукоусиление: знание логарифмической шкалы (дБ), расчёта SPL (SPL = Sens + 10·log(P) - 20·log(D)), времени реверберации по Сэбину (RT60 = 0.161·V / Σ(αᵢ·Sᵢ)), индекса разборчивости STI, подбора громкоговорителей по диаграмме направленности.
  • Видеоконференцсвязь: понимание протоколов SIP/H.323, архитектур MCU и SFU, расчёта битрейта для P2P и многоточечных конференций, учёта задержек и джиттера, настройки QoS для приоритизации RTP-трафика.
  • Эргономика: знание угловых порогов зрения человека (угловая минута), методов расчёта оптимальной зоны видимости (ЗОВ) и коэффициента эргономики Кэ, требований к размерам шрифтов и дистанциям до экранов.
  • Электроснабжение и охлаждение: понимание активной и полной мощности, коэффициента мощности, КПД ИБП, типов аккумуляторов (AGM, GEL, LiFePO₄), учёта пусковых токов, расчёта тепловыделения и подбора кондиционеров.
  • Нормативная база: умение применять ГОСТ Р, SMPTE, IEEE, ISO, AVIXA, BICSI в проектной документации.
  • Проектирование: опыт создания технических заданий, однолинейных схем, спецификаций оборудования, расчёта бюджетов (мощность, охлаждение, ИБП).

Методология расчётов

Все расчёты в Sputnik Studio основаны на признанных инженерных формулах, аппроксимациях, полученных из практических измерений, и отраслевых стандартах. Ниже приведены ключевые методики, используемые в приложении.

Видеопотоки

Битрейт несжатого видео (Мбит/с) = (ширина × высота × FPS × битность × кол-во каналов) / 1 000 000

Для сжатых видеопотоков применяются эмпирические коэффициенты, основанные на типовых профилях кодеков.

LED-экраны

Пиксели_кабинета = floor(размер_кабинета_мм / шаг_мм)
Разрешение_гор = пиксели_кабинета_гор × кол-во_кабинетов_гор
Физическая_ширина_м = кол-во_каб_гор × ширина_каб_мм / 1000
Площадь_м² = ширина_м × высота_м
Мощность_Вт = площадь_м² × удельная_мощность_Вт/м²

Удельная мощность для indoor-экранов принимается 250–300 Вт/м², для outdoor — до 600–800 Вт/м² в зависимости от яркости. Округление пикселей вниз гарантирует целое разрешение кабинета, как у реальных производителей.

Акустика

SPL (дБ) = Sens + 10·log(P) - 20·log(D) + 10·log(N) (для массива)
RT60 (с) = 0.161·V / Σ(αᵢ·Sᵢ)

Эргономика

h = L·tg(α) или L = h / tg(α)

α — угловой размер символа (градусы):
0.25° (14–16 угловых минут) — минимальная читаемость,
0.38° (20–22 угловых минуты) — оптимальные условия,
0.55° (30–35 угловых минут) — критическая информация.

Энергия (ИБП и кондиционер)

Полная мощность (ВА) = (P_активная / cos φ) × запас_пусковые × (1 / КПД_ИБП)
Ёмкость батарей (А·ч) = (P_активная × время_часы) / (напряжение_батареи × коэфф_разряда × КПД_ИБП)
Тепловая нагрузка помещения (BTU/ч) = (удельная_нагрузка_Вт/м² × площадь) / 0.293
Рекомендуемая мощность кондиционера = ближайший стандарт ≥ (тепловыделение_оборудования + тепловая_нагрузка_помещения)

Глоссарий

SPL (Sound Pressure Level) — уровень звукового давления, логарифмическая мера громкости, дБ.

RT60 (Reverberation Time) — время реверберации, с.

PoE (Power over Ethernet) — передача питания по Ethernet: 802.3af (15.4 Вт), 802.3at (30 Вт), 802.3bt (60/90 Вт).

QoS (Quality of Service) — механизм приоритизации трафика (DSCP, CoS).

Multicast — групповая рассылка, оптимизированная IGMP Snooping.

Chroma Subsampling — цветовая субдискретизация (4:4:4, 4:2:2, 4:2:0).

Bit Depth — глубина цвета (8/10/12 бит на канал).

MCU (Multipoint Control Unit) — устройство для многоточечных видеоконференций, микширует потоки.

Latency — задержка прохождения сигнала (мс).

PTP (Precision Time Protocol) — протокол точного времени IEEE 1588 для синхронизации.

VLAN (Virtual LAN) — логическая сегментация сети (802.1Q).

IGMP Snooping — механизм коммутатора для управления Multicast-трафиком.

UPS (Uninterruptible Power Supply) — источник бесперебойного питания.

BTU (British Thermal Unit) — британская тепловая единица, 1 BTU/ч ≈ 0.293 Вт.

Видео

Раздел содержит настройки разрешения, цветовой субдискретизации, частоты кадров, цветового пространства и глубины цвета. Эти параметры влияют на расчёт битрейта видеопотоков в трактах и задержку, вносимую кодексами.

Более подробное описание видеокодеков и цветовых параметров приведено в разделе методологии.

Сеть

Настройки сети (среда передачи, Multicast, QoS, тип сети, синхронизация, резервирование) влияют на расчёт загрузки сети, используемых портов и PoE-бюджета. В виджетах сайдбара отображается суммарный битрейт, загрузка, количество использованных портов, статус Multicast и QoS, а также PoE-бюджет, общая мощность и тепловыделение.

LED-экраны

LED-калькулятор поддерживает два режима:

  • По кабинетам: пользователь задаёт шаг пикселя, размер кабинета и количество кабинетов. Система вычисляет реальное разрешение (пиксели кабинета = floor(размер_мм / шаг_мм) × количество кабинетов), физические размеры, площадь и мощность.
  • По разрешению: пользователь задаёт желаемое разрешение (Full HD, 4K, 8K или своё). Система подбирает ближайшее количество кабинетов вверх, чтобы покрыть заданное разрешение, и выводит реальное разрешение, количество кабинетов, размеры, площадь и мощность.

Такой подход гарантирует целое число пикселей в кабинете, что соответствует реальным LED-изделиям.

Звук

Раздел содержит пять режимов акустических расчётов:

  • SPL на расстоянии: расчёт уровня звукового давления с учётом чувствительности, мощности, расстояния, headroom и помещения.
  • Падение SPL от расстояния: для точечного и линейного источников.
  • Изменение SPL от мощности: изменение уровня при изменении подводимой мощности.
  • RT60 (реверберация): время реверберации по формуле Сэбина.
  • Подбор громкоговорителей: расчёт количества потолочных громкоговорителей по площади помещения, общей мощности и SPL в зоне прослушивания.

ВКС (видеоконференции)

Два режима:

  • Расчёт битрейта кодеков (P2P): рекомендуемый битрейт отправки и приёма для выбранной платформы, разрешения, FPS и количества участников.
  • Многоточечный вызов (MCU): битрейт на одного участника и суммарный битрейт для конференции с заданным количеством участников.

Тракты

Сквозной тракт (сигнальная цепь) — последовательность устройств от источника до приёмника. Позволяет оценить суммарную задержку, энергопотребление, PoE-бюджет и количество задействованных портов.

  • Устройства: источники (ПК, плееры), передатчики (HDMI->HDBaseT), коммутаторы, матрицы, LED-процессоры, экраны.
  • Параметры: задержка (мс) каждого устройства, потребляемая мощность, поддержка PoE, количество портов Ethernet.
  • Расчёт: суммарная задержка (мс и в кадрах при текущем FPS), общая мощность, бюджет PoE.

В приложении реализован конструктор тракта с возможностью добавления/удаления устройств и мгновенным пересчётом.

Эргономика ЭКП

Для экранов коллективного пользования реализованы два калькулятора:

  • Расчёт расстояния/высоты символа: позволяет по заданной высоте символа определить максимальное расстояние до экрана и наоборот, с выбором категории информации (минимальная читаемость, оптимальные условия, критическая информация).
  • Зона оптимальной видимости (ЗОВ): с возможностью добавления участников, заданием координат их глаз, размера экрана и положения строки. Рассчитывается доля строки, попадающая в зону с угловым размером символа не ниже критического, и коэффициент эргономики Кэ (средняя доля × 10).

Методика основана на угловых размерах (α = 0.25°, 0.38°, 0.55°).

Энергия (ИБП и кондиционер)

Модуль позволяет подобрать источник бесперебойного питания и кондиционер на основе суммарной мощности оборудования и тепловыделения, рассчитанных в трактах и LED-калькуляторе.

  • Подбор ИБП: пользователь задаёт время резервирования (часы), выбирает тип ИБП (on‑line или line‑interactive), коэффициент запаса на пусковые токи, напряжение и тип батарей (AGM, GEL, LiFePO₄). Система рассчитывает рекомендуемую мощность ИБП (ВА) и ёмкость батарей (А·ч).
  • Подбор кондиционера: пользователь может указать площадь помещения, высоту потолков и тип помещения (офис, серверная, торговый зал). Тепловая нагрузка помещения суммируется с тепловыделением оборудования, и подбирается ближайший стандартный размер кондиционера (9000, 12000, 18000, 24000, 30000, 36000, 48000, 60000 BTU/ч).

Все настройки сохраняются в браузере и восстанавливаются при следующем открытии.

Логгер событий

Встроенный инструмент для отладки и мониторинга работы системы. Логгер фиксирует:

  • Запуск приложения с информацией об операционной системе, языке, разрешении экрана.
  • Состояние сети (онлайн/офлайн, тип соединения, скорость, RTT) с периодической проверкой (раз в минуту).
  • Действия пользователя: клики по кнопкам, изменение полей ввода, переключение аккордеонов, создание/удаление трактов и т.д.
  • Изменения состояния приложения (смена режима, количество трактов).

Доступ к логам защищён паролем. При первом запуске генерируется случайный пароль, который выводится в консоль (для администратора). В интерфейсе (кнопка «Показать логи» в разделе «УПРАВЛЕНИЕ») при вводе пароля открывается модальное окно со списком событий, возможностью фильтрации по типу и экспортом в JSON. Логи хранятся в localStorage, максимальное количество записей — 500.

Безопасность AV-систем

  • Сетевая сегментация: выделение AV-трафика в отдельные VLAN с запретом меж-VLAN-маршрутизации.
  • 802.1X (Network Access Control): аутентификация устройств перед предоставлением доступа к сети.
  • Шифрование потоков: для критичных данных (ВКС, управление) использовать SRTP или IPsec.
  • Управление доступом: на коммутаторах и матрицах использовать ACL для ограничения управления только доверенными IP-адресами.
  • Парольная политика: смена заводских паролей, использование сложных паролей, централизованная аутентификация (RADIUS, TACACS+).
  • Физическая безопасность: стойки с замками, контроль доступа в серверные.

Практические кейсы

Кейс 1. Ситуационный центр губернатора

Задача: спроектировать видеостену 4K и разместить 12 рабочих мест с оптимальной видимостью.

Решение: использован LED-калькулятор (режим по кабинетам): шаг 1.56 мм, кабинеты 600×337.5 мм, 5×5 кабинетов → разрешение 1920×1080, физический размер 3.0×1.6875 м, площадь 5.06 м², мощность ~1.5 кВт. Эргономика: расчёт зон видимости для 12 наблюдателей дал Кэ = 8.9.

Кейс 2. Конференц-зал на 50 мест

Задача: обеспечить качественное звукоусиление и разборчивость речи.

Решение: объём зала 450 м³, RT60 снижен с 1.5 с до 0.7 с с помощью акустических панелей. Подобраны потолочные громкоговорители с чувствительностью 92 дБ, мощностью 30 Вт; SPL на расстоянии 8 м составил 88.7 дБ.

Кейс 3. Подбор ИБП и кондиционера для серверной

Задача: оборудование серверной потребляет 5 кВт активной мощности. Требуется подобрать ИБП с временем резервирования 2 часа и кондиционер.

Решение: в модуле «Энергия» выбраны: тип ИБП on‑line, напряжение батарей 48 В, тип AGM, запас на пусковые токи 1.2. Рекомендация: ИБП мощностью 9500 ВА, ёмкость батарей ~230 А·ч. Для помещения площадью 50 м², высотой 3 м, тип «серверная»: тепловая нагрузка помещения ~15000 Вт (≈51200 BTU/ч). Суммарное тепловыделение оборудования (17060 BTU/ч) + помещения = ~68260 BTU/ч → рекомендован кондиционер 72000 BTU/ч.

Часто задаваемые вопросы

Вопрос 1: Почему в LED-калькуляторе разрешение получилось нецелым?
Ответ: В версии 6.1.0 используется округление пикселей кабинета вниз (floor), поэтому разрешение всегда целое. Если вы видите дробное разрешение, проверьте, что используете актуальную версию.

Вопрос 2: Как получить доступ к логам?
Ответ: В разделе «УПРАВЛЕНИЕ» нажмите «Показать логи», введите пароль. При первом запуске пароль выводится в консоль браузера (F12). Его можно изменить, очистив localStorage.

Вопрос 3: Можно ли сохранить настройки ИБП и кондиционера?
Ответ: Да, все настройки модуля «Энергия» автоматически сохраняются в localStorage и восстанавливаются при следующем открытии.

Вопрос 4: Как рассчитать кондиционер, если нет площади помещения?
Ответ: Если площадь не указана, учитывается только тепловыделение оборудования. Рекомендуется ввести хотя бы примерную площадь для более точного подбора.

Вопрос 5: Почему при сбросе проекта не сбрасываются настройки модуля «Энергия»?
Ответ: Настройки модуля «Энергия» хранятся отдельно, чтобы не сбрасывать их при каждом сбросе проекта. Если нужно сбросить и их, очистите localStorage вручную (или используйте «Сброс проекта» в браузере).

Управление проектом

Все данные, введённые пользователем, автоматически сохраняются в localStorage браузера. Для надёжности и переносимости предусмотрены следующие функции:

  • Сохранить в браузере — запись текущего состояния проекта в localStorage (выполняется автоматически при изменении параметров, также доступна ручная кнопка).
  • Экспорт JSON — выгрузка проекта в файл .json. Формат включает все разделы: видеотракты, LED-экраны, акустические настройки, тракты и пр.
  • Импорт JSON — загрузка ранее сохранённого проекта. При этом данные перезаписываются, интерфейс обновляется.
  • Печать отчёта — открывает новое окно с форматированным отчётом, содержащим все расчётные параметры и итоговые значения. Отчёт оптимизирован для печати.
  • Сброс проекта — удаление всех данных из localStorage и возврат к настройкам по умолчанию (требуется подтверждение).

Важно: данные не синхронизируются между устройствами. Для переноса проекта используйте экспорт/импорт JSON. Рекомендуется периодически делать резервные копии.
